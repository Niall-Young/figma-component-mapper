# 无命名图层识别规则

## 通用原则

- 不依赖图层名也要能做出基础识别
- 先看文本，再看视觉，再看结构，再看上下文
- 无命名图层不等于低置信度，关键是证据是否充分

## Button 识别

常见信号：

- 可点击热区明显
- 文案居中或近似居中
- 有填充、边框、圆角或 hover 态暗示
- 常位于 footer、toolbar、表单操作区

## Input 识别

常见信号：

- 单行矩形输入容器
- 内部有 placeholder、已有值或前后缀 icon
- 文本左对齐且留有输入内边距

## Select 识别

常见信号：

- 输入容器右侧存在箭头
- placeholder 常为“请选择”
- 点击预期是展开候选项

## DatePicker 识别

常见信号：

- 日期格式文本
- 日历 icon 或右侧选择入口
- 常与筛选区、表单区配套出现

## Dialog 识别

常见信号：

- 中央浮层
- 标题 + 关闭入口 + footer 操作区
- 背景下方存在遮罩或弱化暗示

## Table 识别

常见信号：

- 多列网格
- 表头与数据行结构重复
- 常伴随操作列、状态列、分页

## Pagination 识别

常见信号：

- 页码数字、上下页箭头
- 位于表格底部或列表底部
- 与总数、跳转、每页条数控件相邻

## 输出要求

- 高频对象应尽量输出 `name_dependency: none` 或 `low`
- 如果主要依赖命名才能成立，必须显式降级并说明风险
